# Очередь событий SystemV

![Структура очереди system V в ядре](.\SL2F1.png)

## Лекция

### Атрибуты очереди сообщений

- Имя(целое число)
- UID, GID создание очереди
- UID, GID назначение владельца
- Права доступа к очереди сообщений
- время и индефикаатор процесса, который последний передал сообщение в очередь
- время и индефикаатор процесса, который последний прочитал сообщение
- указатель на сообщение

### Сообщение

- Данные сообщения
- Тип сообщения

### Создание/получение доступа(открытие) очереди сообщений

```c
int msget(ley_t  key, int flog)
// key - интрепритатор(имя) очереди сообщений
// flag - флаг
```
#### Флаги

- IPC_CREAT|IPC_EXCL - гаррантированное создание новой очереди сообщений
IPC_CREAT|0777 - разрешить все

### Отправка(запись) сообщения в очередь

```c
int msgsnd(ind msgfd, cons void* msgPtr, int len, int flag)
// msgfd - целое число, которое плученно от msget
// msgPtr - указатель на переменную типа struct msbuf
// len - длина в байтах поля mtext, на который указывает msgPtr
struct msbuf
{
    long mtype; // тип сообщения
    char mtext[MSGMAx] // сообщение
}
```
IPC_NOWAIT - указание на то, что вызов не является блокирующим

### Получение(чтение) сообщения из очереди

```c
int msgrev(int msgfd, const void* mdgPrt, int len, int mtype, int flag)
// msgfd - целое число, которое полученно от msget
// mdgPrt - указатель на переменную, тип котрой похож на msgbuf
// len - размер, сообщение которое хотим считать
// mtype - тип, сообщения которое хотим считать
// flag - флаги
```

Смысл mtype

- O - считать из очереди самое мтарое значение любого типа
- положительное число - считать из очереди самое старое сообщение заданного типа
- отрицательное число - считать из очереди сообщение, тип которого меньше абсолютного значения mtype или равное ему. Если таких сообщений неколько, то считать самое старое

### Управление очередью сообщений

```c
int msgctl(int msgfd, int cmd, struct msgId_ds *mbufPtr)
// msgfd - целое число, которое полученно от msget
// cmd - команда
// msgId_ds * mbufPt - указатель на структуру с полями, которые описывают очередь
```

IPC_STAT - получение атрибутов очереди
IPC_SET - установление атрибутов очереди
IPC_RMID - удаление очереди

```c
msgctl(msgfd, IPC_RMID, Null) // удаление очереди сообщений
```

### Концепции работы с семафорами

```c
int semget(key_t ley, int num_sem, int flag)
int semop(int semId, struct scmbuf *pBuf, int len)

struct sembuf{
    short sem_num;
    short sem_op;
    short sem_flag;
};
```

Флаг
SEM_UNDO 

```c
int semdl(int semId, int num, int cmd, union semnum arg)

union semun{
    int val; // значение семафора
    struct semid_ds *buf; // управляющие парметры
    ushort *array; // массив значений семафора
}
```

- IPC_STAT - получение информации о работе семафоров
- IPC_SET - модификация управляющих парметров
- IPC_RMID - удаление наборов семафоров
- GETALL - получить текущее значение всех семафоров в наборе
- SETALL - установить все значения семафоров с помощью arg.array
- GETVAL - получить значение семафора с номером num (arg не использ)
- SETVAL - установить самафор со значением num в значение arg.val
