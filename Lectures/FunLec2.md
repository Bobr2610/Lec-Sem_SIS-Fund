# Ключевое слово $const$

- Значение определяется на этапе компиляции.

## Ключевое слово $readonly$

- Работает аналогично $const$, но значение можно присвоить, если оно известно только на этапе выполнения.

## Ключевое слово $static$

- Ведет себя примерно так же, как и в C
- Позволяет определить сущность на уровне типа, а не объекта: $static$-переменная, $static$-конструктор
- Если метод помечен $static$, объект создавать не нужно: вызов идет через тип, а не через экземпляр

## Статический ($static$-)конструктор

Нужен для инициализации статических полей. Его нельзя перегружать. Он вызывается один раз.

## Базовый класс $object$

- Все, что является экземпляром ссылочного типа, наследуется от класса $object$

## Статический метод $Parse$

```csharp
var n = int.Parse("12345");
```

Конвертирует строку в указанный тип и возвращает результат. Если конвертация не удалась, происходит исключительная ситуация.

## Статический метод $TryParse$

```csharp
int n;
int.TryParse("12345", out int n);
```

Возвращает `boolean` в зависимости от успеха или неудачи.

## Исключения

Объект, несущий информацию о каком-то событии. Бывают разного вида:

- Генерируются средой (деление на ноль)
- Попытка чтения из файла, которого нет
- ...

В C#, кроме `try/catch`, есть блок `finally`.
Он выполняется всегда.
Нужен, как правило, для освобождения ресурсов.

## Массивы (Array)

Для создания массива с фиксированным размером используется следующая конструкция:

```csharp
Type[] a1;
```

`a1` — массив элементов типа `Type`.

Пример:

```csharp
int[] aint;
```

Это только объявление ссылки на массив. Нужна инициализация, явная или неявная:

```csharp
aint = new int[10];
```

На уровне типа `array` есть стандартные методы для работы с массивами.

### Метод $Array.Sort(aint)$

Используется для сортировки массива. Определен на уровне типа данных.

### Метод $Clear$

Передается диапазон индексов. Соответствующие элементы будут обнулены.

### Метод $CopyTo$

Копирование элементов из одного массива в другой.

### Метод $Reverse$

Обращение массива.

## Операторы $is$ и $as$

Бинарные операторы.

`is` — проверяет, является ли объект указанным типом.

```csharp
obj is Type
```

Если объект слева приводится к типу `Type`, возвращается истина, иначе ложь.
Использовать не надо, так как ссылку можно получить с оператором `as`.

`as` — выполняет приведение объекта к типу `Type`. Если приведение невозможно, возвращается `null`.

```csharp
obj as Type
```

## Ключевое слово $var$

На этапе компиляции заставляет компилятор определить тип переменной.

Если тип переменной можно определить из контекста, это можно отдать компилятору. `var` является аналогом `auto` из C++.

- `var` нельзя использовать при описании метода
- Для переменной без инициализации использовать нельзя
- `var` разрешается только при создании локальной переменной внутри функции
- Создать поле класса типа `var` нельзя

## Стандартные интерфейсы

### `IClonable`

"Клонирование (создание глубокой копии)"

```csharp
interface IClonable
{
    object Clone();
}
```

В реализации `Clone()` выполняется логика глубокого копирования объекта.

### `IComparable`

"Возможность сравнения"

```csharp
interface IComparable
{
    int CompareTo(object ob);
}
```

Правило сравнения:

0. Объекты равны
1. Объект, который является источником вызова `CompareTo`, больше передаваемого объекта
2. Иначе меньше

### `IEnumerable`

"Перечисляемый"

```csharp
interface IEnumerable
{
    IEnumerable GetEnumerator();
}

interface IEnumerator
{
    bool MoveNext();
    object Current { get; }
    void Reset();
}
```

### `IDisposable`

При реализации метода `Dispose()` мы записываем логику освобождения занятых ресурсов.

## Базовый класс $object$

Виртуальные методы:

- `bool Equals(object obj);` — глубокое сравнение объектов
- `void Finalize();` — аналог деструктора, определяется неявно. Вызывается сборщиком мусора. Принудительно вызвать `Finalize` нельзя
- `int GetHashCode();` — можно использовать для описания или сравнения
- `string ToString();` — см. выше

Невиртуальные методы:

- `Type GetType()`
- `object MemberwiseClone();`

## Garbage collector

Экземпляр сборщика мусора создать нельзя.

Есть специальный тип — `GC`. С помощью него можно запустить сборку мусора:

```csharp
GC.Collect();
```

Всего есть 3 поколения: 0, 1 и 2. Все объекты в приложении разделены на эти поколения.

- Большие объекты почти наверняка отправляются в поколение 2
- Маленькие объекты — в меньшие поколения

Недавно созданные объекты отправляются в поколение 0.
Пережившие одну сборку мусора — в поколение 1.
Если пережили больше — в поколение 2.

Сборка мусора выполняется в фоновом режиме, в отдельном потоке.

Сборка мусора начинается в поколении 0. Освобождается память в поколении 0. Если памяти стало хватать — заканчиваем, иначе идем в поколение 1. И так до второго поколения.

В месте, откуда началась сборка мусора, строится граф достижимости. Достижимые объекты могут использоваться в коде. Недостижимые объекты могут быть освобождены. Этот процесс не моментальный, а постепенный.

## Метод $SuppressFinalize$

Означает, что об освобождении ресурсов позаботились мы и `Finalize` не нужен.
