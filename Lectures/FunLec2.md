# Ключевое слово $const$
- Значение определяется на этапе компиляции. 
# Ключевое слово $readonly$
 - Работает аналогично $const$, но:
	 - Если значение известно становится на этапе выполнения, но неизвестно на этапе компиляции, его можно присвоить. В C++ средств решения этой проблемы нет.
# Ключевое слово $static$
- Примерно ведет себя точно так же, как и в C
- Позволяет определить сущность, которая определяется не на уровне объекта, а на уровне типа данных, то есть может быть $static$-переменная, $static$-конструктор. 
- Если метод помечен $static$, нам не нужно создавать объект, чтобы к нему обратиться, то есть он вызывается через тип, а не объект.
# Статический ($static$-)конструктор
 Нужен для инициализации статических полей. Его нельзя перегружать. Он вызывается один раз.

# Статический метод $Parse$
```C#
var n = int.Parse("12345");
```
Конвертирует нашу строку в указанный тип и возвращает. 
Если конвертация не удалась, происходит исключительная ситуация.
# Статический метод $TryParse$
```C#
int n;
int.TryParse("12345", out int n)
```
Возвращает `boolean` в зависимости от удачи/неудачи.

# Массивы (Array)
Для создания массива с фикс. размером используется след. конструкция:
```C#
Type[] a1;
```
`a1` - массив элементов типа `Type`.
Пример:
```C#
int[] aint;
```
Это лишь объявление ссылки на массив. Нужна инициализация, явная или не явная: 
```C#
aint = new int[10]
```
На уровне типа `array` есть стандартные методы для работы с ними.
## Метод $Array.Sort(aint)$
Используется для сортировки массива. Определен на уровне типа данных.
## Метод $Clear$
Передается диапазон индексов. Соотвествующие элементы будут обнулены.
## Метод $CopyTo$
Копирование элементов из одного массива в другой
## Метод $Reverse$
Обращение массива

# Операторы $is$ и $as$
Бинарные операторы.

$is$ - проверяет, является ли объект таким-то типом.
`obj is Type` 
Если объект слева приводится к типу `Type`, то возвращается истина, иначе ложь.
Использовать не надо, так как ссылку всегда можно получить с оператором `as`. (оператор реализован через `as`).

`obj as Type`
Выполняется приведение объекта к типу Type. Если приведение невозможно, возвращается `null`

# Ключевое слово $var$
На этапе компиляции заставляет компилятор определить тип переменной. 
Для создания переменной надо указать ее тип. Иногда тип переменной может быть определен из контекста. Это задачу можно отдать компилятору. `var` является аналогом `auto` из C++.

Слово `var` нельзя использовать при описании метода. При описании метода тип должен быть указан явно.
Использовать для переменной без инициализации нельзя.
`var` разрешается только при создании локальной переменной у функции. Создать поле класса типа `var` нельзя.

# Garbage collector
Экземпляр сборщика мусора самому создать нельзя
Есть специальный тип - GC. С помощью него можно запустить сборку мусора:
```C#
GC.collect();
```
Всего есть 3 типа поколений: 0, 1 или 2. Все объекты в приложении разделены на эти 3 поколения. 
Большие объекты почти наверное отправляются в поколение 2.
Маленькие объекты - в меньшие поколения. 

Недавно созданные объекты отправляются в поколение 0.
Пережившие одну сборку мусора - в поколение 1.
Если пережили больше - в поколение 2.

Сборка мусора всегда происходит в фоновом режиме. Раньше это была не асинхронная операция, теперь это делается в отдельном потоке. 

Сборка мусора начинается в поколении 0. Освобождается та память, что есть в поколении 0. Если памяти стало хватать - заканчиваем, иначе идем в поколение 1. И так до второго поколения.

В месте, откуда началась сборка мусора, строится граф достижимости. Те объекты, которые достижимы, могут использоваться в коде. Те объекты, которые не видны - они не достижимы и память от них может быть освобождена. Этот процесс не моментальный, а постепенный. Выполняется до тех пор, пока запрос на память от оператора `new` удален не будет.
