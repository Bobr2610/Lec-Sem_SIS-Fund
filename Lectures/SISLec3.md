## Объекты VFS

- Суперблок
- Файловый индекс (информация о файле)
- Элемент каталога
- Открытый файл, который связан с некоторым процессом

В каждом объекте содержатся операции (действия), которые можно выполнить над данным объектом.

## Виртуальные методы VFS

- `super_operations`
- `inode_operations`
- `dentry_operations`
- `file_operations`

### Операции суперблока

```cpp
struct super_operations {
    void (*destroy_inode) (struct inode *);
    int (*write_super) (struct super_block *);
    ...
};
```

```cpp
struct super_block {
    ...
    struct super_operations* s_op;
    ...
};
```

```cpp
struct super_block* sb;
sb->s_op->write_super(sb);
```

## Последовательность шагов функции `open`

1. Ядро выполняет поиск в таблице дескрипторов файлов процесса первой незадействованной позиции. Номер (индекс) этой позиции является возвращаемым значением функции `open`.
2. Ядро выполняет поиск незанятой позиции в таблице открытых файлов ядра.
   - Если свободная позиция найдена, в найденную запись таблицы открытых файлов процесса заносится ссылка на найденную запись таблицы ядра.
   - В таблицу открытых файлов в найденную позицию записывается ссылка на ту запись из таблицы inode, которая соответствует нужному файлу.
   - В таблице открытых файлов индексируется указатель текущей позиции чтения/записи.
   - В запись таблицы открытых файлов записывается режим открытия файла.
   - В записи таблицы открытых файлов изменяется счетчик ссылок на данный файл.
   - Запись счетчика ссылок индексного дескриптора увеличивается на 1.

## Закрытие файла (вызов функции `close()`)

1. Соответствующая запись в таблице открытых файлов процесса помечается как неиспользуемая (незадействованная).
2. Ядром уменьшается значение счетчика ссылок таблицы открытых файлов на 1.
   - Если значение ссылок отлично от 0, перейти к пункту 6.
3. Позиция таблицы открытых файлов помечается как незадействованная.
4. Значение счетчика ссылок в соответствующей записи таблицы индексных дескрипторов файла уменьшается на 1.
   - Если значение счетчика ссылок отлично от 0, перейти к пункту 6.
5. Если значение счетчика ссылок в записи таблицы индексных дескрипторов отлично от 0, то происходит возвращение управления вызвавшему процессу с кодом 0.
   - Иначе ядро помечает соответствующую запись в таблице индексных дескрипторов как неактивную и физически удаляет память, которая выделена на диске.
6. Ядро возвращает управление вызвавшему процессу и код успешного выполнения 0.
